## 项目简介

SmartConvert 是一款基于 Web 的文档格式转换工具，专为现代文档处理需求而设计。该项目致力于为开发者、撰稿人和学生提供一个极简、高效且视觉精美的文档处理平台，支持 Word、PDF、Text 与 Markdown 之间的双向互转。

该工具通过现代化的技术栈实现，结合了响应式前端界面和高性能后端处理引擎，为用户提供无缝的文档转换体验。项目采用全栈架构设计，确保从用户界面到数据处理的完整解决方案。

## 核心目标

SmartConvert 项目的核心目标包括三个主要方面：

### 1. 高保真度格式转换
实现精确的文档格式转换，确保在 Word、PDF、Text 与 Markdown 之间转换时能够最大程度地保持原始文档的结构和样式完整性。

### 2. 现代化用户体验
提供响应式、现代化且符合审美的前端交互体验，包括深色模式支持、拖拽上传、实时预览等功能。

### 3. 高性能后端处理
构建基于 Spring Boot 的高性能后端处理引擎，确保快速、稳定的文档转换服务。

## 技术架构选型

### 前端技术栈
- **框架选择**：Vue 3 (Composition API) 或 React 18，提供现代化的组件开发体验
- **构建工具**：Vite，确保快速的开发和构建速度
- **UI 组件库**：Element Plus (Vue) 或 Ant Design (React)，提供丰富的 UI 组件
- **样式库**：Tailwind CSS，实现灵活的样式定制
- **状态管理**：Pinia (Vue) 或 Redux Toolkit (React)，管理应用状态
- **动画库**：Framer Motion 或 GSAP，增强用户体验

### 后端技术栈
- **核心框架**：Spring Boot 3.x，提供企业级 Java 应用开发基础
- **核心库**：
  - Markdown 解析：flexmark-java，专业的 Markdown 处理库
  - Word 处理：Apache POI，成熟的 Office 文档处理库
  - PDF 处理：itext7 或 OpenHTMLtoPDF，可靠的 PDF 处理方案
  - Pandoc 桥接：可选的系统调用，提供极致转换效果
- **接口文档**：Swagger / Knife4j，完善的 API 文档
- **文件上传**：Spring StandardMultipartHttpServletRequest，标准的文件处理机制

### 部署架构
- **容器化**：Docker & Docker Compose，确保环境一致性
- **服务器**：Nginx，负责前端静态资源路由和反向代理

## 主要功能特性

### 核心转换模块
SmartConvert 提供三种主要的转换路径：

1. **Word ↔ Markdown**：保留标题、列表、表格和加粗等基本样式
2. **PDF ↔ Markdown**：
   - PDF 转 MD：提取文本内容，尽量保持层级结构
   - MD 转 PDF：支持代码高亮渲染后的美化导出
3. **Text ↔ Markdown**：纯文本与 Markdown 格式的封装与去格式化

### 用户界面需求
- **深色模式支持**：提供一键切换深浅色主题
- **拖拽上传**：仿制 Vercel 或 Apple 风格的拖拽上传区域
- **实时预览**：Markdown 转换前/后的实时预览窗口（左编辑右预览）
- **进度反馈**：带有动效的转换进度条（Processing...）
- **批量处理**：支持一次性上传多个文件并打包下载

### 后端 API 功能
- **POST /api/convert**：核心接口，接收文件、源格式、目标格式，返回转换后的文件流或下载链接
- **GET /api/history**：查看最近转换记录（基于 Session 或本地存储）
- **GET /api/health**：系统健康检查

## 设计理念

### 极简主义设计
采用极简白/石墨黑的配色方案，点缀色使用 Indigo (600) 或 Emerald (500)，营造专业而现代的视觉效果。

### 卡片式布局
转换工具箱采用悬浮卡片感，增加 backdrop-blur (毛玻璃) 效果，提供层次分明的界面设计。

### 交互反馈
通过按钮点击时的微互动（Micro-interactions）和上传成功后的 Toast 提示，增强用户的操作反馈。

### 响应式设计
确保在各种设备和屏幕尺寸上都能提供良好的用户体验。
